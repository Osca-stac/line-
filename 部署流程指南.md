# 從 Cursor 到 Vercel 部署完整流程指南

> 適用專案：React + Vite + TypeScript 前端專案  
> 工具鏈：Google AI Studio → Cursor → GitHub → Vercel

---

## 概覽

```
Google AI Studio（原型）→ Cursor（優化）→ GitHub（版本控制）→ Vercel（部署上線）
```

---

## 第一階段：在 Cursor 完成開發

在 Cursor 中完成程式碼的開發與優化後，確認以下幾點：

- 專案根目錄有 `package.json`
- 確認 build 指令正常（可在 Cursor 終端機執行 `npm run build` 測試）
- 確認 `.gitignore` 有排除 `node_modules/` 與 `.env*`

---

## 第二階段：建立 GitHub Repository

### 1. 登入 GitHub

前往 [https://github.com](https://github.com)，登入你的帳號。

### 2. 新增 Repository

1. 點右上角「**+**」→「**New repository**」
2. 填入 Repository name（例如：`line-sticker-cutter`）
3. 選擇「**Public**」（公開）或「Private」（私人）
4. **不要**勾選「Add a README file」（本地已有檔案，勾選會造成衝突）
5. 點「**Create repository**」

---

## 第三階段：將本地程式碼推上 GitHub

### 在 Cursor 內建終端機（或系統 Terminal）執行：

**步驟 1：進入專案資料夾**
```bash
cd 你的專案資料夾路徑
```

**步驟 2：初始化 Git 並提交**
```bash
git init
git add .
git commit -m "first commit"
git branch -M main
```

**步驟 3：設定遠端 GitHub 網址並推送**

> ⚠️ 如果出現 `error: remote origin already exists`，請用 `set-url` 取代 `add`：

```bash
# 第一次設定（沒有 remote 時）
git remote add origin https://github.com/你的帳號/你的repo名稱.git

# 如果 remote 已存在（出現 already exists 錯誤時）
git remote set-url origin https://github.com/你的帳號/你的repo名稱.git
```

```bash
git push -u origin main
```

### 確認成功

推送完成後，前往 `https://github.com/你的帳號/你的repo名稱`，確認頁面上顯示出 `src/`、`package.json`、`index.html` 等檔案。

---

## 第四階段：用 Vercel 部署

### 1. 登入 Vercel

前往 [https://vercel.com](https://vercel.com)，使用 GitHub 帳號登入。

### 2. 授權 Vercel 存取 GitHub Repo

點「**Add New → Project**」，若清單中找不到你的 repo：

1. 點畫面下方的「**Adjust GitHub App Permissions →**」
2. 在 GitHub 頁面找到「Repository access」
3. 選「**Only select repositories**」，從下拉選單加入你的 repo
4. 點「**Save**」
5. 回到 Vercel 頁面重新整理

### 3. Import 並設定專案

1. 找到你的 repo，點「**Import**」
2. 確認以下設定：

| 設定項目 | 應填入的值 |
|---|---|
| Application Preset | **Vite** |
| Build Command | `npm run build` |
| Output Directory | `dist` |
| Root Directory | `./` |

> ⚠️ 若 Application Preset 顯示「Other」，請手動改成「**Vite**」，Vercel 才能正確偵測 build 設定。

3. 如有用到 API Key（如 Gemini API），在「**Environment Variables**」展開填入：
   - Key：`GEMINI_API_KEY`
   - Value：你的 API Key

4. 點「**Deploy**」

### 4. 等待部署完成

約 1～2 分鐘後，Vercel 會提供一個公開網址，格式如：
```
https://你的專案名稱.vercel.app
```

## 常見錯誤排解

### ❌ `error: remote origin already exists`
**原因**：本地 git 已設定過 remote，但指向錯誤網址。  
**解法**：
```bash
git remote set-url origin https://github.com/正確帳號/正確repo名稱.git
```

### ❌ `fatal: repository not found`
**原因**：remote URL 是範例文字（如「你的帳號」）而非真實網址。  
**解法**：同上，用 `git remote set-url` 修正為正確網址。

### ❌ `nothing to commit, working tree clean`
**原因**：檔案已在之前被 commit 過，不影響 push。  
**解法**：直接繼續執行 `git push`。

### ❌ Vercel 錯誤：`repository does not contain the required branch`
**原因**：GitHub repo 還是空的，程式碼尚未推上去。  
**解法**：確認 `git push` 有成功執行，GitHub 頁面上有出現檔案後再回 Vercel 部署。

### ❌ Vercel build 失敗（找不到 dist 或 build error）
**原因**：Application Preset 沒有設定成 Vite。  
**解法**：在 Vercel 專案設定中將 Framework Preset 改為「Vite」，重新部署。

## 後續更新

之後每次在 Cursor 修改程式碼，只需執行：

```bash
git add .
git commit -m "你的更新說明"
git push
```

Vercel 會自動偵測 GitHub 有新的 commit，並**自動重新部署**，不需要手動操作 Vercel。
